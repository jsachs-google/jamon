Jamon

What is Jamon?

Jamon is a template processor for Java, useful for generating dynamic html, dynamic text, basically any dynamic text-based content. The syntax of the templates is based on Mason.

Isn't Jamón a food?

Jamón is a succulent spanish pork product, not to be confused with ham! Nor confused with Jamon.

What isn't Jamon?

Jamon does not provide the functionality of a controller, nor of a servlet. It does one thing, and does it well: it is a template language for Java.

Why is Jamon?

Jamon provides the following advantages over JSP and other technologies:
  • Separate presentation from controller logic
  • General-use; can produce arbitrary output format
  • Not tied to HTML or XML (like Enhydra XMLC)
  • Pass arbitrary objects between template [Tea]
  • Invocation of templates is typed like any other method call
  • Can pass arbitrary dynamic content (i.e. template fragments)
